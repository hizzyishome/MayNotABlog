### 链表及其变体的特点
![链表类型和优势](./链表类型和优势.png)  
大概的一个意思，其实说来就是，双向便于回溯找到某节点前一个节点；带头节点（也叫哨兵节点）便于处理边界节点逻辑，统一代码逻辑；循环从任意一个节点开
都可以遍历整个链表。当然越复杂占用空间越多，但是能大幅度优化时间复杂度，这点空间付出是很值得的。

#### 1. linkFirst
```
    // train
    public void linkFirst(T t){
        Node<T> tNode = new Node<T>(t);
        if (head != null) {
            Node<T> item = head;
            head = tNode;
            head.next = item;
        } else {// empty list
            head = tNode;
            tail = tNode;
        }
        size++;
    } 
```

```
    // JDK
    private void linkFirst(E e) {
        final Node<E> f = first;
        final Node<E> newNode = new Node<>(null, e, f);
        first = newNode;
        if (f == null)
            last = newNode;
        else
            f.prev = newNode;
        size++;
        modCount++;
    }
```
分析：　
- 在jdk中搞成private，不对外暴露这个方法，应该是基于链表通用方法名考虑，减少有歧义的方法入口
- jdk中节点初始化是双向链表节点的构造函数，将节点前后关系在构造时候维护代码更简洁，
更合理
- 关于modCount，在其父类AbstractList中有如下注释
```
    /**
     * The number of times this list has been <i>structurally modified</i>.
     * Structural modifications are those that change the size of the
     * list, or otherwise perturb it in such a fashion that iterations in
     * progress may yield incorrect results.
     *
     * <p>This field is used by the iterator and list iterator implementation
     * returned by the {@code iterator} and {@code listIterator} methods.
     * If the value of this field changes unexpectedly, the iterator (or list
     * iterator) will throw a {@code ConcurrentModificationException} in
     * response to the {@code next}, {@code remove}, {@code previous},
     * {@code set} or {@code add} operations.  This provides
     * <i>fail-fast</i> behavior, rather than non-deterministic behavior in
     * the face of concurrent modification during iteration.
     *
     * <p><b>Use of this field by subclasses is optional.</b> If a subclass
     * wishes to provide fail-fast iterators (and list iterators), then it
     * merely has to increment this field in its {@code add(int, E)} and
     * {@code remove(int)} methods (and any other methods that it overrides
     * that result in structural modifications to the list).  A single call to
     * {@code add(int, E)} or {@code remove(int)} must add no more than
     * one to this field, or the iterators (and list iterators) will throw
     * bogus {@code ConcurrentModificationExceptions}.  If an implementation
     * does not wish to provide fail-fast iterators, this field may be
     * ignored.
     */
    protected transient int modCount = 0;
```
    1. modCount为list结构修改次数。
    2. 结构修改 -> 改变大小 或者在迭代过程中可能返回错误结果的操作（也就是多线程状态下
    对当前集合的不同步修改，比如你在迭代遍历某一集合过程中，另一个线程增删改了这个集合）
    3. iterator 和 listIterator 会用到，遍历过程中该值如果变化，
    抛出ConcurrentModificationException
    4. fail-fast 机制是java集合(Collection)中的一种错误机制。当多个线程对同一个
    集合的内容进行操作时，就可能会产生fail-fast事件。
    例如：当某一个线程A通过iterator去遍历某集合的过程中，若该集合的内容被其他线程所
    改变了；那么线程A访问集合时，就会抛出ConcurrentModificationException异常，
    产生fail-fast事件。
    5. 在子类中是可选实现的，如果在迭代过程中需要实现“快速失败”机制，add和remove需要
    对该值做修改。

#### 3. linkBefore
```
    // train
    public void linkBefore(T t, Node<T> item) {
        Node<T> tNode = new Node<T>(t);
        if (item != null) {
            Node<T> prev = item.prev;
            if (prev == null) {// item is head node
                linkFirst(t);
            } else {
                prev.next = tNode;
                tNode.prev = prev;
                tNode.next = item;
            }
        } else {
            throw new NullPointerException("Node item is null");
        }
        size++;
    }
```

```
    // JDK
    void linkBefore(E e, Node<E> succ) {
        // assert succ != null;
        final Node<E> pred = succ.prev;
        final Node<E> newNode = new Node<>(pred, e, succ);
        succ.prev = newNode;
        if (pred == null)
            first = newNode;
        else
            pred.next = newNode;
        size++;
        modCount++;
    }
```
分析：　
- 我写的多了判空，而jdk中这个作为底层操作，在上层调用时候做的判空。这里也启发我们，
基础操作和逻辑操作最好有个分离，这个对于设计工程方面有很重要的意义。

#### 4. unlinkFirst
```
    // train
    public T unlinkFirst(Node<T> f){
        T val = f.val;
        if (f != null) {
            head = f.next;
            f.val = null;
            f.next = null;
        } else {
            throw new NullPointerException("Node is null");
        }
        return val;
    }
```

```
    // JDK
    private E unlinkFirst(Node<E> f) {
        // assert f == first && f != null;
        final E element = f.item;
        final Node<E> next = f.next;
        f.item = null;
        f.next = null; // help GC
        first = next;
        if (next == null)
            last = null;
        else
            next.prev = null;
        size--;
        modCount++;
        return element;
    }
```
分析：　
- 设计到删除操作的时候，虽然java内存管理交给了虚拟接，但是不当的内存占用仍然可能会导致
OOM(OutOfMemory) Error f.next = null; 后面的help GC 和精妙，因为这里不是触发GC
而是对GC有帮助而已，是否进行GC不是这里能决定的。[(相关资料)](https://blog.csdn.net/levena/article/details/78318830)

```
    // JDK
    E unlink(Node<E> x) {
        // assert x != null;
        final E element = x.item;
        final Node<E> next = x.next;
        final Node<E> prev = x.prev;

        if (prev == null) {
            first = next;
        } else {
            prev.next = next;
            x.prev = null;
        }

        if (next == null) {
            last = prev;
        } else {
            next.prev = prev;
            x.next = null;
        }

        x.item = null;
        size--;
        modCount++;
        return element;
    }
```
分析：

#### 10. clear
```
    // JDK
    /**
     * Removes all of the elements from this list.
     * The list will be empty after this call returns.
     */
    public void clear() {
        // Clearing all of the links between nodes is "unnecessary", but:
        // - helps a generational GC if the discarded nodes inhabit
        //   more than one generation
        // - is sure to free memory even if there is a reachable Iterator
        for (Node<E> x = first; x != null; ) {
            Node<E> next = x.next;
            x.item = null;
            x.next = null;
            x.prev = null;
            x = next;
        }
        first = last = null;
        size = 0;
        modCount++;
    }
```

- 这里说清空链表中所有

     * Removes all of the elements from this list.

    public boolean addAll(int index, Collection<? extends E> c) {
        checkPositionIndex(index);

        Object[] a = c.toArray();
        int numNew = a.length;
        if (numNew == 0)
            return false;

        Node<E> pred, succ;
        if (index == size) {
            succ = null;
            pred = last;
        } else {
            succ = node(index);
            pred = succ.prev;
        }

        for (Object o : a) {
            @SuppressWarnings("unchecked") E e = (E) o;
            Node<E> newNode = new Node<>(pred, e, null);
            if (pred == null)
                first = newNode;
            else
                pred.next = newNode;
            pred = newNode;
        }

        if (succ == null) {
            last = pred;
        } else {
            pred.next = succ;
            succ.prev = pred;
        }

        size += numNew;
        modCount++;
        return true;
    }

    private void checkPositionIndex(int index) {
        if (!isPositionIndex(index))
            throw new IndexOutOfBoundsException(outOfBoundsMsg(index));
    }

    private boolean isPositionIndex(int index) {
        return index >= 0 && index <= size;
    }
```

- 添加都是把集合对象通过toArray方法转化成数组，然后添加

    // train
    Node node(int index){
        if (index < 0 || index >= size ) {
            return null;
        }
        Node e = head;
        for (int i = 0; i < index; i++) {
            e = e.next;
        }
        return e;
    }


    //JDK
    Node<E> node(int index) {
        // assert isElementIndex(index);

        if (index < (size >> 1)) {
            Node<E> x = first;
            for (int i = 0; i < index; i++)
                x = x.next;
            return x;
        } else {
            Node<E> x = last;
            for (int i = size - 1; i > index; i--)
                x = x.prev;
            return x;
        }
    }
```
分析：　
- 这里在查找的时候用到了双向链表的特性，先二分，看Node在前半还是后半。
- JDK中很多出除法运算都是用位移操作代替了，网上怎么说的都有，有的说直接*更快，有的说
位移更快。这个要涉及到操作系统，和操作系统位数也有关系。编译成汇编代码后，看机器指令
是如何表达的。普遍从效率上看，使用移位指令有更高的效率，因为移位指令占2个机器周期，
而乘除法指令占4个机器周期。从硬件上看，移位对硬件更容易实现，所以会用移位，
移一位就乘2,这种乘法当然考虑移位了。
